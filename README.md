# HA Custom Schedule Card

Home Assistant의 순정 `schedule` 헬퍼를 시각적으로 관리할 수 있는 커스텀 카드입니다.

단일 JavaScript 파일로 구성되어 있으며, 외부 의존성 없이 동작합니다.
사용자의 Home Assistant 언어 설정에 따라 한국어와 영어 인터페이스가 자동으로 전환됩니다.

## 주요 기능

* **단일 파일 구성** — `schedule-ui.js` 하나로 설치와 관리가 간단합니다. HACS 커스텀 저장소 등록도 지원합니다.
* **1카드 1스케줄** — 카드당 하나의 스케줄 엔티티를 매핑하여 직관적으로 시간 블록을 편집할 수 있습니다.
* **24시간 타임라인** — 등록된 시간 블록을 24시간 바 형태로 시각화하여 일정을 한눈에 파악할 수 있습니다. 현재 시각 표시선도 포함되어 있습니다.
* **매일 탭** — 7개 요일의 공통 시간 블록(교집합)을 '매일' 탭에서 한 번에 확인하고 편집할 수 있습니다.
* **스케줄 활성/비활성** — 카드 상단의 토글 버튼으로 스케줄 자체를 켜거나 끌 수 있습니다.
* **다국어 지원** — `hass.language` 감지를 통해 한국어(오전/오후)와 영어(AM/PM) 시간 표기가 자동으로 적용됩니다.
* **다크 테마 최적화** — 글래스모피즘 스타일의 다크 UI로 설계되어, Home Assistant의 다크 테마에 자연스럽게 어울립니다.

## 설치

### HACS를 통한 설치 (권장)

1. HACS 메뉴를 엽니다.
2. 우측 상단 메뉴에서 **사용자 지정 저장소**를 선택합니다.
3. 저장소 URL에 아래 주소를 입력합니다.

   ```
   https://github.com/jewon-oh/schedule-ui
   ```

4. 카테고리를 **Lovelace**로 설정하고 추가합니다.
5. 목록에서 `Custom Schedule Card`를 찾아 다운로드합니다.
6. 대시보드 리소스 자동 추가를 확인하는 팝업이 나타나면 승인합니다.

### 수동 설치

1. `schedule-ui.js` 파일을 Home Assistant의 `/config/www/` 디렉토리에 복사합니다.
2. **설정 → 대시보드 → 리소스**에서 `/local/schedule-ui.js`를 `JavaScript Module` 유형으로 추가합니다.

## 사용 방법

### 1. 스케줄 헬퍼 생성

카드를 사용하기 전에 Home Assistant에서 스케줄 헬퍼를 먼저 만들어야 합니다.

1. **설정 → 기기 및 서비스 → 도우미**로 이동합니다.
2. **도우미 만들기 → 일정(Schedule)**을 선택합니다.
3. 이름과 아이콘을 설정하고 생성합니다. 시간 블록은 카드에서 추가하면 되므로 비워두어도 괜찮습니다.
4. 생성된 엔티티 ID를 확인합니다. (예: `schedule.livingroom_light`)

### 2. 카드 추가

대시보드에서 카드를 추가할 때, 비주얼 에디터에서 설정하거나 YAML을 직접 입력할 수 있습니다.

```yaml
type: custom:ha-custom-schedule-card
entity: schedule.livingroom_light      # 스케줄 엔티티 ID (필수)
toggle_entity: switch.livingroom       # 연결할 기기 엔티티 ID (선택)
title: 거실 전등 일정                   # 카드 제목 (선택)
```

| 옵션 | 필수 | 설명 |
|------|------|------|
| `entity` | O | 스케줄 헬퍼의 엔티티 ID. `schedule.*` 도메인만 지원합니다. |
| `toggle_entity` | X | 스케줄과 연동할 기기의 엔티티 ID. 스위치, 조명, 팬, 에어컨 등을 선택할 수 있습니다. |
| `title` | X | 카드 상단에 표시할 제목. 미입력 시 스케줄의 이름이 자동으로 표시됩니다. |

## 브릿지 자동화 설정

스케줄 헬퍼는 현재 시각이 등록된 시간 블록에 해당하면 `on`, 아니면 `off` 상태를 반환하는 역할만 합니다.
실제로 조명이나 스위치 같은 기기를 제어하려면, 스케줄의 상태 변화를 감지하여 기기를 켜고 끄는 **자동화**가 필요합니다.

```
schedule.livingroom_light → on  →  자동화  →  switch.livingroom: turn_on
schedule.livingroom_light → off →  자동화  →  switch.livingroom: turn_off
```

`examples/` 디렉토리에 두 가지 예제가 포함되어 있습니다.

| 파일 | 설명 |
|------|------|
| `automation-bridge.yaml` | 수동으로 자동화를 작성하는 예시입니다. |
| `schedule-bridge-blueprint.yaml` | 블루프린트로 등록하면 여러 스케줄-기기 쌍에 재사용할 수 있습니다. |

### 블루프린트 사용 절차

1. `schedule-bridge-blueprint.yaml` 파일을 Home Assistant의 `/config/blueprints/automation/` 하위 폴더에 복사합니다.
2. **설정 → 자동화 → 블루프린트**에서 해당 블루프린트를 선택합니다.
3. 스케줄 엔티티와 대상 기기를 지정하면 자동화가 생성됩니다.

## 로컬 미리보기 (개발용)

Home Assistant 없이 카드의 UI를 확인하고 싶다면 `preview.html`을 브라우저에서 열어볼 수 있습니다.

```bash
python -m http.server 8080
# http://localhost:8080/preview.html 접속
```

언어 선택 드롭다운을 통해 한국어/영어 간 인터페이스 전환을 확인할 수 있습니다.
